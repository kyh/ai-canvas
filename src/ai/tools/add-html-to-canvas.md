Use this tool to add a generated HTML block to the canvas. This tool takes the HTML/CSS/JS code generated by the `generateHTML` tool and creates an HTML block node on the canvas.

## When to Use This Tool

Use Add HTML To Canvas when:

1. You have generated HTML code using the `generateHTML` tool
2. You need to add an interactive HTML element to the canvas
3. The user wants to place HTML content on the canvas

## Workflow

1. First, call `generateHTML` to analyze the image - this creates a loading placeholder block and returns its ID
2. Then, call `addHTMLToCanvas` **exactly once** with:
   - The generated HTML
   - **REQUIRED**: The `updateBlockId` from the `generateHTML` response (you MUST include this)
3. **CRITICAL**: 
   - You MUST provide the `updateBlockId` - it is required, not optional
   - Call this tool exactly ONCE per user request
   - Do NOT call it multiple times
   - Once you've added the HTML, the task is complete

## Required Properties

- **html**: The complete HTML string (including `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`, embedded CSS and JS)
- **label**: Descriptive name for the block (e.g., "Interactive Button", "Form Component")
- **x, y**: Position on the canvas (should match the loading block position)
- **width, height**: Dimensions of the HTML block (should match the loading block dimensions)
- **visible**: true (default)
- **opacity**: 0-100 (100 = fully opaque)
- **updateBlockId**: **REQUIRED** - ID of the loading block created by `generateHTML`. You MUST include this ID to update the loading block. Do not omit this parameter.

## Positioning Logic

### If Selection Exists (selectionBounds provided):

Position the HTML block **to the right** of the selected blocks:
- **x**: `selectionBounds.x + selectionBounds.width + spacing` (use 20-40px spacing)
- **y**: `selectionBounds.y` (align with top of selection, or center vertically: `selectionBounds.y + (selectionBounds.height - height) / 2`)

### If No Selection:

Position the HTML block at the **center of the canvas**:
- Canvas default size: 1280x720
- **x**: `(1280 - width) / 2` (approximately 640 - width/2)
- **y**: `(720 - height) / 2` (approximately 360 - height/2)

## Dimensions

**You must determine appropriate width and height** based on the generated HTML content:

1. **Analyze the HTML structure** to understand the natural size
2. **Consider the content**: 
   - Buttons: typically 100-200px wide, 40-60px tall
   - Forms: typically 300-500px wide, height depends on fields
   - Cards/Containers: match the visual design from the image
3. **Use reasonable defaults** if uncertain:
   - Minimum: 200x100
   - Typical: 400x300
   - Maximum: 800x600

## Block Properties

- **type**: Must be `"html"`
- **html**: The complete HTML string with embedded CSS and JS
- **background**: Optional background color (hex string, e.g., "#FFFFFF")
- **border**: Optional border object with `color`, `width`, and `dash` array
- **radius**: Optional corner radius object with `tl`, `tr`, `br`, `bl` values
- **shadow**: Optional shadow object with `color`, `offsetX`, `offsetY`, `blur`, `enabled`
- **rotation**: Optional rotation in degrees (default: 0)
- **scaleX, scaleY**: Optional scaling factors (default: 1)
- **opacity**: 0-100 (default: 100)
- **visible**: true (default)

## Best Practices

- **Determine dimensions thoughtfully**: Analyze the HTML content to set appropriate width/height
- **Position relative to selection**: If selection exists, place to the right with proper spacing
- **Center if no selection**: Use canvas center coordinates
- **Match visual design**: Ensure the HTML block dimensions match what was shown in the image
- **Include all styling**: The HTML should be self-contained with embedded CSS
- **Include interactions**: The HTML should include JavaScript for any interactive elements

## Important Notes

- **Call this tool only once** per HTML generation task
- Once you've called it with the complete HTML, do not call it again
- The tool will update the loading block with your HTML
- If you call it multiple times, only the first update will be applied (subsequent calls will be ignored)

## Summary

Use Add HTML To Canvas to place generated HTML code on the canvas. Call it exactly once with the complete HTML and the `updateBlockId` from the loading block. Do not call it multiple times.
